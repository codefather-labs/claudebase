## Conventions

- **`Verification Class`** — one of `CLI`, `DB`, `Mixed`, `FS`. Most Wave A tests are `CLI` or `Mixed` (CLI + DB) per ba-analyst's verification-class hints in use-cases file lines 11–13. No UI/UX cases (no browser surface in Wave A).
- **`Evidence Required`** — concrete artifact names under `docs/qa/evidence/claudebase-v0.9/`. Vague entries ("works correctly", "no errors") are auto-FAIL at qa-engineer execution time.
- **SQL command** — `sqlite3 <path> "<query>"` where `<path>` is the full absolute path to the relevant DB. Local DB: `<cwd>/.claude/knowledge/insights.db`. Global DB: `~/.claude/knowledge/insights.db` (or `C:\Users\madwh\.claude\knowledge\insights.db` on Windows). Substitute concrete path at runtime.
- **Exit-code capture** — PowerShell: `$LASTEXITCODE`; Bash: `$?` / `echo $?`. The qa-engineer captures the actual integer and compares it to the expected value.
- **stderr capture** — PowerShell: redirect via `2>&1` or `$ErrorStream`; Bash: `2>stderr.txt`.
- **Evidence storage** — all artifacts live under `docs/qa/evidence/claudebase-v0.9/` (directory created by Slice 10). The qa-engineer creates the directory at first run.
